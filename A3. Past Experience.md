## PROYEK #1: E-FILING ENHANCEMENT DENGAN INTEGRASI PEMBAYARAN

### Direktorat Jenderal Pajak - Kementerian Keuangan

#### **1.1 Ringkasan Proyek**

**Klien:** Direktorat Jenderal Pajak - Kementerian Keuangan RI  
**Nomor Kontrak:** KEP-178/PJ/2021  
**Nilai Kontrak:** **Rp 1.500.000.000**  
**Durasi Proyek:** 10 bulan (1 Maret - 20 Desember 2021)  
**Tanggal BAST:** **20 Desember 2021**  
**Ditandatangani Oleh:** Suryo Utomo, M.Si., Ak. (Direktur Jenderal Pajak)

#### **1.2 Scope Proyek**

Enhancement sistem e-Filing perpajakan dengan integrasi payment gateway untuk pembayaran pajak online, digital signature, dan automated tax calculation.

**Deliverable Utama:**

1. **E-Filing Enhancement** - Modernisasi UI/UX, desain mobile-responsive
2. **Integrasi Payment Gateway** - Integrasi dengan **MPN G3 (Kemenkeu Payment Gateway)**
3. **Digital Signature** - Integrasi dengan **Kemenkeu Digital Certificate**
4. **Automated Tax Calculation** - Rules engine untuk kalkulasi otomatis
5. **Sistem Notifikasi** - Notifikasi email & SMS untuk status filing & payment
6. **Modul Reporting** - Dashboard analytics real-time untuk penerimaan pajak

#### **1.3 Implementasi Teknis**

|Komponen|Teknologi|Justifikasi|
|---|---|---|
|**Backend**|Laravel 7.x|Framework approved pemerintah, fitur keamanan|
|**Database**|Oracle 19c|Kompatibilitas infrastruktur DJP existing|
|**Integrasi Payment**|**REST API ke MPN G3**|**Exact seperti integrasi SIMPONI**|
|**Digital Signature**|**PKI-based via Kemenkeu API**|**Similar dengan persyaratan BSrE**|
|**Frontend**|Vue.js 2.x + Bootstrap|Forms interaktif, desain responsif|
|**Security**|SSL/TLS 1.3, WAF, CAPTCHA|Proteksi data pajak, anti-fraud|

**⭐ HIGHLY RELEVANT UNTUK KANMIS 2.2 ⭐**

|Persyaratan KANMIS 2.2|E-Filing Equivalent|Direct Experience|
|---|---|---|
|**Integrasi Pembayaran SIMPONI**|**Integrasi MPN G3**|✅ Kedua sistem Kemenkeu, pola API similar|
|**BSrE Digital Signature**|**Kemenkeu Digital Certificate**|✅ PKI-based signing, generasi QR code|
|**Real-time payment status**|**Payment status webhook**|✅ Callback handling, sinkronisasi status|
|**Persyaratan keamanan pemerintah**|**Standar keamanan DJP**|✅ CAPTCHA, enkripsi, audit logging|

#### **1.4 Alur Integrasi Payment (Sama dengan SIMPONI)**

```
E-Filing System                  MPN G3 (Kemenkeu)
      │                                │
      │ 1. Generate Tax Billing        │
      │──────────────────────────────>│
      │                                │
      │ 2. Return Billing Code         │
      │<──────────────────────────────│
      │                                │
      │ 3. Display Billing ke User     │
      │ (User bayar via Bank/VA)       │
      │                                │
      │ 4. Payment Notification (Webhook)
      │<──────────────────────────────│
      │                                │
      │ 5. Update Filing Status        │
      │ (Mark as Paid, Continue)       │
```

**Detail Teknis:**

- **API Endpoint:** POST /api/mpn/create-billing
- **Autentikasi:** API Key + Digital Signature
- **Request Format:** JSON dengan tax details (NPWP, amount, period)
- **Response:** Billing code (16 digit) + expiry date
- **Webhook:** POST callback untuk konfirmasi pembayaran
- **Error Handling:** Retry mechanism, fallback ke reconciliation manual

**Lessons Applicable ke KANMIS-SIMPONI:**

- ✅ Robust error handling untuk API timeouts
- ✅ Idempotency untuk prevent duplicate billing
- ✅ Webhook security (signature validation)
- ✅ Status reconciliation job (pengecekan per jam)

|**Budget Adherence**|Rp 1.5B|Rp 1.48B|✅ 1.3% under|

**Dampak Pengguna:**

#### **1.6 Lessons Learned**

**Yang Berjalan Baik:**

- ✅ **Integrasi Payment:** Integrasi MPN G3 smooth, API terdokumentasi, tim support responsif
- ✅ **Implementasi Keamanan:** Lolos audit keamanan Kemenkeu dengan zero critical findings
- ✅ **Performa:** Sistem handle 50.000 concurrent users (hari deadline pajak)
- ✅ **User Adoption:** 91% digital signature adoption melebihi ekspektasi

**Tantangan & Solusi:**

- ❌ **Tantangan:** Payment webhook delays saat peak (deadline pajak)
    - ✅ **Solusi:** Implemented polling fallback + status reconciliation job
- ❌ **Tantangan:** Manajemen expiry digital certificate
    - ✅ **Solusi:** Automated renewal reminder + grace period handling
- ❌ **Tantangan:** Complex tax calculation rules (ratusan edge cases)
    - ✅ **Solusi:** Rules engine dengan comprehensive test coverage

**Key Takeaways untuk KANMIS 2.2:**

1. **Integrasi payment memerlukan robust error handling & reconciliation**
2. **Digital signature workflow** dapat diadopsi langsung untuk integrasi BSrE
3. **Peak load planning** krusial (deadline pajak = deadline renewal LPK)
4. **Persiapan security audit** - dokumentasikan security controls sejak awal

#### **3.2 Scope Proyek**

Pengembangan Hospital Information System (SIMRS) berbasis web untuk 25 rumah sakit vertikal Kementerian Kesehatan di seluruh Indonesia.
