## 2. METODOLOGI PROJECT MANAGEMENT

### 2.1 Pendekatan Waterfall Berbasis PMBOK

**Rujukan:** Schwalbe Chapter 2 - The Project Management and IT Context

> "The project management framework provides a basic structure for understanding project management" (Schwalbe, p.60)

Untuk proyek KANMIS 2.2, kami mengadopsi **metodologi Waterfall dengan framework SDLC** sesuai **Metodologi KAK (Halaman 6-7)**:

```
1. ANALISIS KEBUTUHAN (Hari 1-21)
   ├─ Review desain KANMIS / User Requirement
   ├─ Analisis hasil review desain KANMIS
   └─ Dokumentasi requirements

2. DESAIN (Hari 22-42)
   ├─ Desain database
   ├─ Desain alur aplikasi  
   ├─ Desain UI/UX
   └─ Desain infrastructure

3. PENGEMBANGAN (Hari 43-117)
   ├─ Pengembangan backend (Laravel)
   ├─ Pengembangan frontend (Bootstrap)
   ├─ Implementasi database
   ├─ Integrasi SIMPONI
   └─ Implementasi fitur keamanan

4. PENGUJIAN (Hari 107-132)
   ├─ User Acceptance Test (UAT)
   ├─ Performance testing
   ├─ Security testing (Pentest)
   └─ Bug fixing

5. DEPLOYMENT (Hari 127-150)
   ├─ Setup server
   ├─ Migrasi database
   ├─ Deployment aplikasi
   ├─ Pelatihan
   └─ Dokumentasi

6. PEMELIHARAAN (6 bulan post-BAST)
   └─ Bug fixing & support
```

**Rasionalisasi:** Waterfall cocok untuk KANMIS 2.2 karena:

- Requirements terdefinisi baik dalam KAK
- Pengadaan pemerintah memerlukan scope tetap
- Milestone jelas untuk jadwal pembayaran
- Kepatuhan dengan standar SDLC pemerintah

### 2.2 Implementasi PMBOK Knowledge Areas

Sesuai **Schwalbe Chapter 1 - Introduction to Project Management**, kami mengimplementasikan 10 PMBOK Knowledge Areas:

#### **Integration Management**

- Pengembangan **Project Charter** dengan approval PPK
- **Project Management Plan** komprehensif (semua subsidiary plans)
- **Change Control Board** untuk perubahan scope
- Dokumentasi **Lessons Learned**

#### **Scope Management**

- **Requirements Collection** via wawancara stakeholder
- **WBS Development** berorientasi deliverable
- **Scope Validation** penerimaan formal per milestone
- **Scope Control** prosedur change request

#### **Schedule Management**

- Analisis **CPM (Critical Path Method)**
- **Gantt Chart** dengan Microsoft Project
- **Milestone Tracking** review progress mingguan
- **Schedule Performance** monitoring SPI

#### **Cost Management**

- **Budget Baseline:** Rp 919.024.500 fixed-price
- **Earned Value Management (EVM)** tracking bulanan
- **Cost Control** analisis variansi (threshold ±10%)
- **Payment Milestones** sesuai KAK

#### **Quality Management**

- **Quality Planning:** Proses ISO 9001
- **Quality Assurance:** Code reviews, protokol testing
- **Quality Control:** Defect tracking, acceptance testing
- **Continuous Improvement:** Siklus PDCA

#### **Resource Management**

- **Team Acquisition:** Rekrutmen per persyaratan KAK
- **Team Development:** Pelatihan & skill building
- **Resource Allocation:** Kejelasan RACI matrix
- **Performance Management:** Evaluasi berbasis KPI

#### **Communications Management**

- Pemeliharaan **Stakeholder Register**
- **Communication Plan** (laporan harian, mingguan, bulanan)
- **Status Reporting** kepada PPK (Habib Nurhasan)
- **Meeting Management** agenda terstruktur

#### **Risk Management**

- **Risk Register** 20+ risiko teridentifikasi
- **Risk Assessment** probability & impact matrix
- **Risk Response** strategi mitigasi
- **Risk Monitoring** review bulanan

#### **Procurement Management**

- Keputusan **Make-or-Buy** terdokumentasi
- **Vendor Selection** kriteria & evaluasi
- **Contract Management** pengawasan subkontraktor
- **Procurement Closure** proses BAST

#### **Stakeholder Management**

- **Stakeholder Identification** pemetaan komprehensif
- **Engagement Strategy** pendekatan customized
- **Expectation Management** komunikasi proaktif
- **Stakeholder Satisfaction** survei

### 2.3 Framework Quality Assurance

**Rujukan:** Schwalbe Chapter 8 - Project Quality Management, p.318

> "Quality is the degree to which the project fulfills requirements"

**Proses Kualitas:**

```
PLAN QUALITY
├─ Definisikan standar kualitas (ISO, persyaratan BSN)
├─ Tetapkan kriteria penerimaan (per KAK)
├─ Desain strategi testing
└─ Dokumentasikan metrik kualitas

PERFORM QUALITY ASSURANCE
├─ Audit proses (internal & eksternal)
├─ Kepatuhan quality checklists
├─ Peer code reviews
├─ Design reviews
└─ Implementasi best practices

CONTROL QUALITY
├─ Eksekusi testing (unit, integration, UAT)
├─ Tracking & resolusi defect
├─ Acceptance testing dengan BSN
├─ Performance benchmarking
└─ Validasi keamanan
```

**Metrik Kualitas:**

|Metrik|Target|Metode Pengukuran|
|---|---|---|
|**Kualitas Kode**|80%+ test coverage|Automated testing|
|**Defect Density**|< 5 defects/KLOC|Sistem defect tracking|
|**User Acceptance**|90%+ satisfaction|Survei UAT|
|**Performa**|< 3s page load|Load testing|
|**Keamanan**|Zero critical vulns|Penetration testing|
|**Ketersediaan**|99.5% uptime|Monitoring tools|
